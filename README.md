## 快速开始

### 1. 克隆仓库

```bash
git clone https://github.com/your-org/scripts-fifa-world-cup2026.git
cd scripts-fifa-world-cup2026
```

### 2. 安装依赖

如果项目基于 Node.js 管理脚本，建议使用：

```bash
npm install
```

或

```bash
pnpm install
```

### 3. 初始化本地环境

根据仓库中的配置模板复制环境文件：

```bash
cp .env.example .env
```

然后按需填写接口地址、资源路径、构建参数或发布配置。

### 4. 运行常用命令

```bash
npm run dev
npm run build
npm run lint
npm run test
```

如果仓库中包含 `scripts/` 目录，可进一步执行特定脚本，例如：

```bash
node scripts/fetch-schedule.js
node scripts/optimize-assets.js
node scripts/generate-meta.js
```

> 提示：具体命令名称可根据你的项目实际脚本调整，建议统一在 `package.json` 中维护，方便团队成员快速查阅。

## 使用说明

本仓库适合作为世界杯主题项目的“基础工具层”使用，常见方式包括：

### 1. 用于新项目初始化
在创建 FIFA World Cup 世界杯 2026 专题或活动页时，可以直接复用已有目录结构、脚本模板与基础配置，减少从零搭建的时间成本。建议优先建立以下模块：

- `scripts/`：存放自动化脚本
- `data/`：存放赛程、球队、场馆等结构化数据
- `assets/`：管理图片、图标、视频封面等静态资源
- `docs/`：记录规则、接口说明、发布流程

### 2. 用于数据与资源处理
赛事相关项目常涉及大量素材与文本更新，例如队名修正、赛程批量导入、图片压缩、SEO 信息生成等。这类工作非常适合通过脚本自动化完成，避免手工操作导致的遗漏与重复。

建议优先将以下任务脚本化：

- 批量重命名素材文件
- 清洗 JSON / CSV 数据
- 统一页面元信息
- 检查资源引用是否失效
- 生成构建前检查报告

### 3. 用于团队协作规范
当多人同时参与开发时，规范比工具更重要。建议在仓库内明确以下约定：

- 分支命名规则，如 `feature/schedule-page`
- 提交信息规范，如 Conventional Commits
- 发布流程说明
- 环境变量管理方式
- 素材与数据更新责任人

### 4. 用于持续集成
如果项目需要频繁迭代，可接入 GitHub Actions 实现自动检查，例如：

- Push 后自动执行 lint/test
- 合并前自动校验构建
- 发布时自动生成产物
- 定时同步外部数据源

这样可以让项目在世界杯相关节点密集更新时，依然保持较高的交付稳定性。🚀
